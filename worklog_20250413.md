## tool-research-index-init 实现

按 plan 做了完整的论文筛选 skill，支持两种策略：

- **Embedding 语义检索**：用 Qwen3-Embedding-8B 对 47255 篇做了全量 embedding，缓存为 npz 文件（456MB，4096 维）
- **关键词 + LLM 两阶段**：先关键词粗筛到 ~500 篇，再 LLM 逐篇打分

### embedding 构建过程踩了不少坑

1. sentence-transformers 5.4.0 引入了 torchcodec 依赖，服务器缺 FFmpeg，卸了 torchcodec 解决
2. 单卡加载 8B FP16 模型后 batch 推理爆显存
3. 改成 device_map="auto" 4 卡 pipeline parallel，能跑但效率很低（串行流转，每张卡一半时间在等）
4. 最终方案：bitsandbytes INT8 量化（~8GB/卡）+ 4 卡数据并行，每卡独立处理 1/4 数据，122 papers/s，6.5 分钟跑完
5. mp.Manager().dict() 传大矩阵会损坏，改成每个 GPU 子进程独立保存 npy 分片，主进程合并

### 增量更新和自检

加了 `--check-cache` 自检（shape/NaN/norm/重复ID/一致性）和 `--incremental` 增量更新（只 encode 新增论文合并到现有缓存）。

### query encoding

M1 8GB 跑不了 8B 模型，query encoding 通过 SSH 调服务器上的 encode_query.py，返回 JSON 向量，本地纯 numpy 做余弦检索。

## 两种筛选方法对比测试

用 "4D Gaussian Splatting" 方向做了端到端测试，用 subagent 直接打分（省去部署 LLM 服务）：

| | Embedding | Keyword |
|---|---|---|
| S 级命中 | 17/50 (34%) | 12/50 (24%) |
| S+A 精准率 | 23/50 (46%) | 15/50 (30%) |
| 重合 | 3 篇 | 3 篇 |

结论：embedding 精准率更高，但两种方法互补性很强（重合极少），合并后 S 级论文能到 25+ 篇。

## ResearchFlow 对比

对比了 resmax 和 ResearchFlow 的文献搜集能力。resmax 在大规模结构化采集上更强（自动化数据源发现 + 专用 parser），ResearchFlow 在采集后的全链路（PDF 分析 → 知识库 → 查询）更完整。
